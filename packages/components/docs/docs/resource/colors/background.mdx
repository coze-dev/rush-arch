# 背景色 background

背景色是用户界面的基础层，为整个应用提供视觉基调。合理的背景色选择可以提高内容的可读性，减轻视觉疲劳，并为其他UI元素提供良好的衬托。

## 使用原则

- **视觉舒适**: 背景色应当柔和，不应造成视觉刺激或疲劳
- **内容衬托**: 背景色应为前景内容提供良好的衬托，增强可读性
- **主题一致**: 背景色应与整体设计主题保持一致，支持品牌调性

## 应用场景

- 页面主体背景
- 应用整体背景
- 大面积内容区域
- 模态框背景
- 不同层级的容器背景

## 颜色分类

### 主题色 (Theme)

主题色是与产品主题相关的背景色调，包括亮色模式和暗色模式下的不同层级背景。

## 深色模式考量

在设计深色模式时，背景色的选择尤为重要。深色背景应当：

- 避免纯黑色（#000000），以减少对比度过高导致的视觉疲劳
- 保持适当的色调，避免灰度过高导致界面沉闷
- 与前景色保持足够对比度，确保内容可读性

## 背景色层级

背景色通常按照以下层级组织：

- **最强层级 (bg-max)**: 用于弹窗浮层、导航栏、卡片和面板
- **强层级 (bg-plus)**: 用于全页面浮层、导航栏和弹窗
- **默认层级 (bg-primary)**: 用于全页面背景
- **弱层级 (bg-secondary)**: 用于编辑区等特殊区域
