# 阴影色 shadow

阴影色用于创建界面元素的立体感和层次感，通过模拟光影效果增强用户界面的空间感知。合理使用阴影可以提升界面的质感和深度，帮助用户理解界面的层级结构。

## 使用原则

- **层次表达**: 阴影应清晰表达元素的层级关系，高层级元素阴影更明显
- **一致性**: 保持阴影使用的一致性，避免同一层级使用不同阴影
- **适度使用**: 避免过度使用阴影，以免造成视觉混乱

## 应用场景

- 卡片和面板阴影
- 弹出层和模态框
- 下拉菜单
- 悬浮按钮
- 交互反馈（如点击、悬停状态）

## 颜色分类

### 主题色 (Theme)

主题色是与产品主题相关的阴影色调，适用于不同层级和状态的阴影效果。

## 阴影层级

阴影通常按照以下层级组织：

- **大阴影 (shadow-large)**: 用于高层级元素，如模态框、重要卡片等
- **默认阴影 (shadow-default)**: 用于常规元素，如普通卡片、下拉菜单等
- **小阴影 (shadow-small)**: 用于低层级元素，如按钮、提示框等

## 阴影参数

阴影效果通常由以下参数组成：

- **颜色**: 阴影的颜色，通常是半透明的黑色或主题色
- **偏移量**: 阴影在水平和垂直方向的偏移
- **模糊半径**: 阴影的模糊程度
- **扩散半径**: 阴影的扩散范围

## 深色模式考量

在深色模式下，阴影的使用需要特别注意：

- 降低阴影的不透明度
- 可能需要使用亮色阴影而非暗色阴影
- 考虑使用内阴影（inset shadow）增强层次感
